操作符对 （=+， =- 或 =! ） 的使用顺序被反转， 单个操作符（+=， -= 或 !=） 会被编译并运行， 但不会产生预期的结果。

当=+， =-， 或 =!在两个操作符之间没有任何间距的情况下使用，且后面至少有一个空格字符时，此规则会引发一个问题。

不合规的代码示例

target = -5
num = 3

target =- num # 不合规； target = -3。这真的是要表达的意思吗？
target =+ num # 不合规； target = 3

合规的解决方案

target = -5
num = 3

target = -num # 合规；分配num逆值的意图很明显
target += num
