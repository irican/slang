当两种方法具有相同的实现时，要么是错误的——有其他意图——或者是故意重复的，但这可能会让维护者感到困惑。在后一种情况下， 代码应该被重构。

不合规的代码示例

class MyClass {
  fun calculateCode(): String {
    doTheThing()
    doOtherThing()
    return "done"
  }

  fun getStatus(): String {  // 不合规
    doTheThing()
    doOtherThing()
    return "done"
  }
}

合规的代码示例

class MyClass {
  fun calculateCode(): String {
    doTheThing()
    doOtherThing()
    return "done"
  }

  fun getStatus(): String = calculateCode()
}

例外

忽略少于2个语句的方法。
